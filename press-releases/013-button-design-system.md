# Business Event Matrix Adopts the AgileDataGuides Button Design System

**27 March 2026**

AgileDataGuides today released a unified button design across the Business Event Matrix, adopting the same two-tier button pattern used by every app in the AgileDataGuides Pattern Template family.

## The Problem

The original buttons were a mix of styles — some pastel-filled, some bordered, some plain. Visually it worked, but there was no consistent rule for which style meant what, and the BEM didn't match the visual language of the Information Product Canvas or the other apps in the family. Users hopping between apps had to relearn what each button colour meant.

## The Solution

Every button in the Business Event Matrix now follows the AgileDataGuides two-tier pattern:

- **Object Create buttons** use a pastel-coloured background with a `+` prefix — e.g. `+ Event`, `+ Domain`, `+ Concept`. They mean "add a new entity to the matrix".
- **Canvas Action buttons** use a white background with a coloured border and matching text — e.g. **New Matrix**, **Delete**, **Export JSON**, **Import JSON**. They mean "do something with the current matrix as a whole".

## Key Benefits

- **Predictable across apps** — the same button pattern means the same thing in every AgileDataGuides app
- **Style follows function** — pastel-fill for create, outline for canvas-level actions
- **Consistent across modes** — the same button styling renders in both standalone and Context Plane embedded modes

The unified button design is available now in the latest release.
