# Business Event Matrix Adds Domain Owner Field

**24 March 2026**

AgileDataGuides today released the Domain Owner field for the Business Event Matrix, giving teams a place to record who is responsible for each domain in their model.

## The Problem

Domains exist because someone is accountable for them. But the matrix had no place to capture that accountability — users were either dropping owner names into the description (clunky), into a notes field that didn't exist yet (impossible), or into a separate spreadsheet (drift guaranteed).

## The Solution

Each domain now has an Owner field. The owner can be a person ("Jane Smith"), a team ("Sales Operations"), or a role ("Customer Success Manager"). It shows up in the domain's hover tooltip alongside the description and aliases, and is editable from the Domain Details modal.

## Key Benefits

- **Accountability captured in the model** — no separate spreadsheet to drift
- **Visible at hover** — see the owner in two seconds without opening the modal
- **Free-form** — accept person, team, or role names equally
- **Saved with the matrix** — the owner persists alongside everything else in the JSON file

Domain owners are available now in the latest release.
